\section{Sky Culture Maker Plugin}
\label{sec:plugins:SCM}
\sectionauthor*{Luca-Philipp Grumbach, Susanne M. Hoffmann}

\subsection{Introduction}
\label{sec:plugins:SCM:Introduction}

To ease the creation of sky cultures (Chapter~\ref{ch:SkyCultures}) and avoid mistakes caused by the shortcomings of manual coding, Stellarium in 2025 started developing a ``Sky Culture Maker'' (SCM) plugin.
It provides the technical means of visually creating a new sky culture, allowing the user to interact with the usual Stellarium star map while drawing constellation stick figures and attaching artwork in a wizard-like editor.
The SCM also prompts the user to supply copyright and category information, geographical schema, and all other requirements for a well-defined sky culture.

With the many new sky culture features released in 2025 (a new data format, display options for historical frames of reference, and audio output of sky culture information) the requirements on both the data files and the accompanying description have grown a lot.
The SCM is intended to handle these details for the user and provide an intuitive UI.

However, beware that the plugin is still in development.
It was originally set up as a student project in a software development course, but tests with students from the humanities and researchers in cultural astronomy showed that it is already useful for users who are comfortable editing source code.
Future versions will continue to improve the SCM.

\subsection{Getting Started}
\label{sec:plugins:SCM:GettingStarted}

\begin{figure}[ht]
\centering\includegraphics[width=0.35\columnwidth]{scm_dialog_start.png}
\caption{SCM plugin: start dialog}
\label{fig:plugins:SCM:DialogStart}
\end{figure}

The SCM plugin is disabled by default.
In order to use it, you will need to activate it first according to section~\ref{sec:Plugins:EnablingPlugins}.
Once activated, a toolbar button \guibutton{0.5}{bt_SCM_Off.png} will appear.
This button is also an indicator of whether the SCM is currently open: pressing it toggles the icon to a glowing version and opens the start dialog shown in Figure~\ref{fig:plugins:SCM:DialogStart}.

The \textit{Create} button opens the main dialog where you can build a new sky culture.
If the Sky Culture Converter was enabled at build time, a \textit{Convert} button also appears (see section~\ref{sec:plugins:SCM:Converting} for details).
The \textit{Edit} button opens a file chooser where you can select an existing sky culture. It will be loaded into the SCM for editing.

\subsection{Converting an Existing Sky Culture}
\label{sec:plugins:SCM:Converting}

The SCM includes a UI for the \textit{Sky Culture Converter} (section~\ref{ch:SkyCultures:converter}) that transforms legacy sky cultures using the old \file{.fab} format into the new JSON-based format (Chapter~\ref{ch:SkyCultures}).

\subsubsection*{Build-time requirement}
The converter is not built by default.
To enable it, pass \texttt{-DSCM\_SHOULD\_ENABLE\_CONVERTER=ON} to CMake when configuring the build.
The converter requires Qt~6 or later and it is automatically disabled for Qt~5 builds and Windows ARM64 targets.
It also depends on \href{https://www.html-tidy.org/}{Tidy HTML}, \href{https://www.gnu.org/software/gettext/}{GNU gettext}, and \href{https://www.zlib.net}{zlib}. 
CMake will download the converter library automatically via CPM.

\subsubsection*{Using the converter}
When the converter has been built, a \textit{Convert} button appears in the SCM start dialog alongside the \textit{Create} button (Figure~\ref{fig:plugins:SCM:DialogStartConverter}).

\begin{figure}[ht]
\centering
\begin{tabular}{cc}
\includegraphics[height=30mm]{scm_dialog_start_converter.png} &
\includegraphics[height=30mm]{scm_dialog_converter_file.png} \\
(a) Start dialog with \textit{Convert} button & (b) Converter file chooser
\end{tabular}
\caption{SCM plugin: converter workflow dialogs}
\label{fig:plugins:SCM:DialogStartConverter}
\end{figure}

\begin{enumerate}
\item Press \textit{Convert} in the start dialog.
\item A file chooser opens (Figure~\ref{fig:plugins:SCM:DialogStartConverter}, panel b). Select an archive that contains the old sky culture files.
Supported formats: \texttt{.zip}, \texttt{.rar}, \texttt{.tar}, \texttt{.7z}.
Selecting a plain directory is not supported by the converter.
Note that only plain \texttt{.tar} archives are supported. 
Compressed variants such as \texttt{.tar.gz} or \texttt{.tar.bz2} are not.
\item The converter processes the archive and writes the resulting files directly into Stellarium's \directory{skycultures} directory.
If the application does not have write permission there, a dialog prompts you to choose an alternative output folder.
\end{enumerate}

\noindent\textbf{Note:} Sky culture identifiers are case-sensitive.
For example, \texttt{chinese\_chenzhuo} and \texttt{chinese\_Chenzhuo} are treated as different cultures.

\subsection{Creating a New Sky Culture}
\label{sec:plugins:SCM:CreatingSkyCulture}

After pressing \textit{Create} in the start dialog, you will be greeted by the \textit{Overview} tab of the main dialog (Figure~\ref{fig:plugins:SCM:TabsOverviewDescriptionCommonNamesLocation}, panel a).
This dialog is different from most dialogs in Stellarium, as it is an independent window that can be moved outside the Stellarium application, including onto a second monitor for more workspace.
To resize it on Windows, you must drag the tiny square handle in the bottom-right corner.

\noindent\textbf{Note:} Because the dialog is an independent window, it may open \emph{behind} the Stellarium main window and therefore not be immediately visible.
If nothing seems to have happened after pressing \textit{Create}, the dialog has most likely opened in the background or on another monitor.
Try clicking once on the Stellarium main window to bring the SCM dialog to the front.
If that does not work, try minimizing the Stellarium main window.
The same behaviour applies to the \textit{Constellation Editor} described in section~\ref{sec:plugins:SCM:CreatingConstellation}.
In the first text input, you may enter a name for your new sky culture. This field is mandatory.
Below the name field is the constellation list, which is empty until you create constellations (see section~\ref{sec:plugins:SCM:CreatingConstellation}).
Before doing that, it is worth browsing the remaining tabs.

\paragraph{Description tab}
The \textit{Description} tab (Figure~\ref{fig:plugins:SCM:TabsOverviewDescriptionCommonNamesLocation} b) collects textual information about the sky culture.
It contains dropdown menus as well as mandatory (marked with an asterisk) and optional text fields.
Scroll through all fields and supply as much information as possible.
Your inputs are used to generate the sky culture description file (\file{description.md}), which Stellarium will use to display certain information to users.
For details on the expected content and structure of this file, see section~\ref{sec:skycultures:description}.

\paragraph{Common Names tab}
The \textit{Common Names} tab (Figure~\ref{fig:plugins:SCM:TabsOverviewDescriptionCommonNamesLocation} c) manages culture-specific names for stars, planets, and deep-sky objects.

The table at the top lists the common name entries you have already created.
The input fields below are used to enter new names.
In order to automatically fill in the type, identifier, and English name, select an object in Stellarium and press \textit{Copy Data From Selected Object}.
The mandatory fields are marked with an asterisk, just like in the Description tab.

Note that in some cultures, planets carry different names depending on the time of day.
The \textit{Visible} dropdown lets you choose whether a name is always visible, or only during morning or evening.
For non-planetary objects the value is fixed at \textit{Always visible}.

For the full specification of common names for stars, planets and deep-sky objects, see sections~\ref{sec:skycultures:starnames}, \ref{sec:skycultures:planetnames}, and~\ref{sec:skycultures:dsonames}.

\noindent Workflow for managing entries:
\begin{itemize}
\item \textbf{Add:} Fill in the fields and press \textit{Save As New Entry}. The fields are cleared afterwards.
\item \textbf{Edit:} Select a row in the table; the fields are populated with the current values. Make your changes and press \textit{Update Selected Entry}.
\item \textbf{Remove:} Select a row and press \textit{Remove Selected Entry}. This action is irreversible!
\end{itemize}

\paragraph{Location tab}
The \textit{Location} tab (Figure~\ref{fig:plugins:SCM:TabsOverviewDescriptionCommonNamesLocation}, panel d) specifies where and when the sky culture was or is practiced.

The upper part of the tab shows a world map, which allows you to digitize the geographic territory of the culture by drawing polygons.
Simply click on the map to register corners and extend the polygon.
\begin{itemize}
\item At least one polygon is required before exporting the sky culture. You can later edit or refine the GeoJSON format in your GIS applications.
\item Hold \keys{\shift} and drag with the left mouse button to move the map while drawing.
\item Press \keys{\ctrl+Z} to undo the last click.
\item Press \keys{Esc} to discard the entire current polygon.
\item Double-right-click to finish the polygon. A dialog then asks for the start and end year during which the culture was or is practiced. The special value \texttt{9146} indicates that the culture is still practiced.
\item A short help summary for all map interactions is available by clicking the \textbf{?} button in the bottom-right corner of the map.
\end{itemize}

\noindent\textbf{Instruction for creating geo-polygons:}
The polygons are meant to indicate the origin of the astronomical culture that is represented in Stellarium.
We regard this information as educational in terms of understanding the reasons for the selection of specific images, because the specific fauna and flora, and the geological features of the area influence cultural habits (astronomy is used for forecasting seasonal weather, encounters with migrating birds, fish, and other animals that might be depicted in the constellations).
For that reason, the polygons of historical indigenous settlements or the extent of empires are more important than modern borders of states or enclosures.
For instance, in a modern state (like the USA) there might be reserves for indigenous nations (e.g.\ the Navajo), but for our purpose these legal boundaries are less important than the original area where their sky cultures had been developed in pre-Columbian America.

\noindent\textbf{Note the Time Slider!}
You have the option to simulate the movement of a culture over time.
For instance, the Indian Vedic culture developed in the Indus Valley first, then spread to the Ganges Plateau and then to the entire Indian subcontinent (and with Hinduism even beyond).
So, we defined three time steps for the Indian Vedic culture with different geographical extents.
This may help in understanding both where any specific culture is practiced today and where it stems from.

\noindent Note that the time slider does not affect the display in the Stellarium main simulation, i.e.\ the display of the sky.
It is only meant to illustrate the evolution of cultures.

The polygon data is then saved to a \file{territory.geojson} file.
For details on its format see section~\ref{sec:skycultures:territory}.
For information on geoscheme regions, see section~\ref{sec:skycultures:region}.

\subsection{Creating a New Constellation}
\label{sec:plugins:SCM:CreatingConstellation}

Head back to the \textit{Overview} tab and press \textit{Add Constellation} to open the \textit{Constellation Editor} (Figure~\ref{fig:plugins:SCM:ConstellationEditorPanels}, panel a). 
This editor consists of three tabs. 


\paragraph{Drawing tab}
The \textit{Drawing} tab lets you draw constellation lines directly on the Stellarium sky map (see also section~\ref{sec:skycultures:constellations} for the underlying data format).

\begin{enumerate}
\item Press the \textit{Pen} button to activate drawing mode.
\item Hover over a star and a snapping indicator will show you which star is currently targeted.
\item Right-click on the star to set it as the start of a line segment.
\item Move the mouse to the next star; right-click again to complete the segment and immediately begin the next one from that point.
\item To stop drawing, double-right-click or simply press the \textit{Pen} button again.
\end{enumerate}

\noindent Using this workflow you can build complex stick figures (see Figure~\ref{fig:plugins:SCM:DrawingArtwork} a).
Mistakes can be undone with the \textit{Undo} button (one segment at a time) or erased with the \textit{Eraser} tool: 
press \textit{Eraser}, then hold the right mouse button and drag through any line segment to remove it.

\paragraph{Information tab}
The \textit{Information} tab (Figure~\ref{fig:plugins:SCM:ConstellationEditorPanels} b) records metadata about the constellation.
The English name and the unique identifier are mandatory; all other fields are optional but recommended.
A possible identifier is generated automatically from the English name, but you may change it manually as well.

\paragraph{Artwork tab}
The \textit{Artwork} tab (Figure~\ref{fig:plugins:SCM:ConstellationEditorPanels} c) lets you attach an illustration to the constellation.
For background on artwork and its role in sky cultures, see section~\ref{sec:skycultures:artwork}.

Press \textit{Upload Image} and select an image file.
Three blue anchor points appear on the image.
To align the artwork with the sky:
\begin{enumerate}
\item Select an anchor point in the tab.
\item Select a star in Stellarium (for example, one that appears in the constellation).
\item Press \textit{Bind Star} to associate that star with the anchor point.
\item Drag the anchor point to the position in the image that the star represents.
\item Repeat for the remaining anchor points.
\end{enumerate}
\noindent An example result is shown in Figure~\ref{fig:plugins:SCM:DrawingArtwork}.
If the image is not visible on the sky map, try repositioning the anchor points.
The SCM does not draw images that become too distorted.

\paragraph{Dark constellations}
The \textit{Overview} tab also offers \textit{Add Dark Constellation}.
The dark constellation editor works just like the regular editor, except that lines are not snapped to stars.
You may click anywhere in the sky to place a vertex using its sky coordinates directly.
See section~\ref{sec:skycultures:constellations} for details on dark constellations.

\paragraph{Managing constellations}
When you save a constellation, it is added to the list in the \textit{Overview} tab (Figure~\ref{fig:plugins:SCM:TestConstellations}).
Select a constellation in the list to \textit{Edit} or \textit{Remove} it.

\subsection{Exporting the Sky Culture}
\label{sec:plugins:SCM:Exporting}

When the sky culture is ready, select a license from the dropdown menu in the \textit{Overview} tab.
For guidance on which license to choose, see section~\ref{sec:skycultures:licenses}.
Then press \textit{Export Sky Culture}.
If any mandatory fields are incomplete, an error message identifies what is missing.
Otherwise the \textit{Export Dialog} (Figure~\ref{fig:plugins:SCM:DialogExport}) opens.

\begin{figure}[ht]
\centering\includegraphics[width=0.85\columnwidth]{scm_dialog_export.png}
\caption{SCM plugin: Export dialog}
\label{fig:plugins:SCM:DialogExport}
\end{figure}

Read through the checklist items and confirm you have followed them.
The checkbox \textit{Merge constellation lines into polylines on export} (checked by default) merges consecutive line segments that share an endpoint: for example, \texttt{[1,2]} and \texttt{[2,3]} become \texttt{[1,2,3]} in the JSON output, producing a cleaner file.
It is recommended to leave this option enabled.

Choose \textit{Export} to export while keeping the SCM open, or \textit{Export and Exit} to export and close the plugin.
The export creates a folder containing:
\begin{itemize}
\item \file{index.json} --- see section~\ref{sec:skycultures:index.json}
\item \file{description.md} --- see section~\ref{sec:skycultures:description}
\item \file{territory.geojson} --- see section~\ref{sec:skycultures:territory}
\item \file{illustrations/} --- sub-folder with any uploaded artwork images
\end{itemize}

\subsection{Closing the Plugin}
\label{sec:plugins:SCM:Closing}

Press the \textit{X} icon in a dialog's title bar or \textit{Cancel} wherever available.
This opens the dialog shown in Figure~\ref{fig:plugins:SCM:DialogClose}, offering three choices:

\begin{figure}[ht]
\centering\includegraphics[width=0.75\columnwidth]{scm_dialog_close.png}
\caption{SCM plugin: Close dialog}
\label{fig:plugins:SCM:DialogClose}
\end{figure}

\begin{description}
\item[Hide editor] Hides all SCM windows but preserves all data. Reopening the SCM via the toolbar button restores the previous state.
\item[Close without saving] Shuts down the SCM plugin and discards all unsaved data.
\item[Don't close] Returns to the SCM without any changes.
\end{description}

\subsection*{Section \big[SkyCultureMaker\big] in config.ini}
\label{sec:plugins:SCM:ConfigIni}

The following settings can be adjusted manually in the \file{config.ini} file under the \texttt{[SkyCultureMaker]} group.
Colors are given as comma-separated R,G,B values in the range 0--1.

\begin{longtable}{l|l|l}\toprule
\emph{Key} & \emph{Type} & \emph{Default} \\\midrule
fixedLineColor        & float R,G,B & \ccbox{1.0,0.5,0.5} \\
fixedLineAlpha        & float       & 1.0                  \\
floatingLineColor     & float R,G,B & \ccbox{1.0,0.7,0.7} \\
floatingLineAlpha     & float       & 0.5                  \\
maxSnapRadiusInPixels & int         & 25                   \\
mergeLinesOnExport    & bool        & true                 \\\bottomrule
\end{longtable}

\noindent\textbf{Key descriptions:}
\begin{description}
\item[fixedLineColor / fixedLineAlpha] Color and opacity of constellation line segments that have been committed (drawn with right-click).
\item[floatingLineColor / floatingLineAlpha] Color and opacity of the preview line segment following the cursor while drawing.
\item[maxSnapRadiusInPixels] Maximum distance in screen pixels within which the cursor snaps to the nearest star or DSO.
\item[mergeLinesOnExport] Whether consecutive line segments sharing an endpoint are merged into polylines on export (mirrors the checkbox in the export dialog).
\end{description}

\subsection{Further Development}
The plugin is still in active development.
In its current state it provides the fundamental features but is not yet fully functional (beta).
If you test the wizard, please report missing features or bugs via the Stellarium issue tracker.
